# YouTube配信ログ生成プロンプト v3（ログ特化・GitHub公開用）

## このプロンプトの目的

字幕・ライブチャット・通常コメント・動画情報の4種のデータを統合し、**情報を最大限保持したアーカイブログ**としてマークダウン形式に整形する。要約・省略は最小限に抑え、後からの検索・引用・AI学習・ファン活動の一次資料として機能することを優先する。

---

## ⚠️ 事前訂正辞書（字幕の誤認識を補正する）

以下の置換を字幕テキスト処理時に適用すること。字幕に限った補正であり、ライブチャット・コメントには適用しない。

| 誤認識（字幕上の表記） | 正しい表記 | 補足 |
|----------------------|-----------|------|
| 秋先生 / 脇先生 | あき先生 | 開発者名 |
| 洋つべ / よつべ | ようつべ | YouTubeのキャラ的な言い換え（意図的表現のためそのまま残す） |
| 地味内 | Gemini | Google AIの名称 |
| ジェミナ | Gemini | 同上 |
| 同説 | 同接 | 同時接続数の略 |
| 安人同腐 | （文脈不明・要確認） | [※要確認] と注記して原文保持 |
| エびチり | エビチリ | 料理名 |

> **辞書の追加方法**：新たな誤認識を発見したらこのテーブルに行を追加する。

## 入力データの種類

1. **字幕ファイル（.srt）**：動画の自動生成字幕（SRT形式）
2. **ライブチャットファイル（\_live\_chat.txt）**：配信中のリアルタイムコメント
3. **通常コメントファイル（\_comments.txt）**：配信後に投稿された通常コメント
4. **動画情報ファイル（\_info.txt）**：タイトル・チャンネル情報・URL等

---

## セクション①：動画情報ヘッダー

`_info.txt` の内容をもとに以下の形式で出力する。

```markdown
# 【動画タイトル】

> **URL**: https://www.youtube.com/watch?v=XXXXXXX  
> **動画ID**: XXXXXXX  
> **配信日**: YYYY-MM-DD  
> **チャンネル**: チャンネル名  
> **説明欄**:
> （説明欄の内容をそのまま記載）
```

## セクション②：統合タイムライン（字幕 + ライブチャット）

字幕とライブチャットは**同一の時刻軸**を持つため、1本の統合タイムラインとして出力する。

### 2-1. SRTのノイズ除去ルール

SRTファイルには以下のノイズが含まれる。**すべて除去**すること。

- 行頭の連番（`1`, `2`, `3` …）
- `align:start position:0%` などのメタデータ
- `<00:00:00,000><c>テキスト</c>` 形式のインライン単語タイミングタグ（`<c>`タグ）
- 前のブロックの末尾行が次のブロックの先頭行に**重複して現れる**パターン（重複行のみ削除し、最後の出現を残す）
- スペースのみ・空白のみの行
- 内容が空のタイムスタンプブロック（`-->` の前後が空のもの）
- `[音楽]` タグ（音楽区間として認識、タイムスタンプのみ `[音楽]` と表記して残す）

### 2-2. 字幕テキストの再構築ルール

- ノイズ除去後、**句点（。）または文脈の自然な区切り**を基準に断片を接続し、発話単位として再構築する
- 事前訂正辞書の置換を適用する
- 明らかに自動認識の失敗と判断できる語句（文脈が完全に破綻している箇所）のみ `[※要確認:原文「〇〇」]` と注記して保持する。**削除しない**

### 2-3. 字幕のタイムスタンプ採用ルール（ログ優先）

**ログ目的のため、タイムスタンプは密に採用する。** 以下の基準で採用する：

- 発話が途切れる区切り（無音・`[音楽]`・沈黙）ごとに1エントリー
- 同一の発話が続く場合は**概ね30〜60秒に1回**を目安にタイムスタンプを打つ
- 話題・場面の切り替わり（新しいマシュマロ読み始め・BGM変化など）は必ずタイムスタンプを打つ
- タイムスタンプ形式：`[MM:SS]` または `[HH:MM:SS]`（1時間未満は前者）

### 2-4. ライブチャットのノイズ除去ルール

| 種別 | 対処 |
|------|------|
| 放送前の待機コメント（タイムスタンプが負の値 `[-X:XX]` のもの） | **全て省略** |
| 絵文字・記号のみのコメント（単発） | 省略 |
| 1〜2文字のみのコメント（単発）（例：「草」「ｗ」「あ」「！」） | 省略。ただし**同一または類似のリアクションが10件以上短時間に集中**している場合は集約して記録 |
| 同一または酷似したコメントの連投（例：「パンパカパンパカポー」「こんにちは」「おつしず」） | 代表1件＋件数で集約（例：`多数「パンパカパンパカポー」（20件以上）`） |
| URLのみ・スパムと判断できるもの | 省略 |

**採用するコメントの基準**（ログ目的のため積極的に残す）：

- 配信内容への**具体的な反応・感想・ツッコミ**（笑いを誘うコメント・ボケへの反応など）
- 配信者への**質問**
- 場の空気・盛り上がりを示すコメント（同一話題への集中）
- 配信内容を補足・解説するコメント
- 固有名詞・出来事・ハプニングへの言及
- ユニークな視点・特徴的な言い回し
- V2では省略されがちだった**中程度の反応コメント**も積極的に採用する

### 2-5. ライブチャットのユーザー名処理

- **ユーザー名（@〇〇）は全て削除**する
- コメント本文のみを記録する
- 例：`@じーてん: 草` → 省略 / `@SIGNAL-ev1ht: ポン酢ASMRｗｗ` → `ポン酢ASMRｗｗ`
- **ただし字幕（発言録）に登場するユーザー名・ID**（配信者がコメントを読み上げた際に発言として記録されているもの）はそのまま保持する
  - 例：「けろちゃん、スパチャありがとうございます」→ 「けろちゃん」はそのまま残す
  - 例：「ミッドキャットさん、無言スパチャ感謝」→ 「ミッドキャットさん」はそのまま残す

### 2-6. 統合タイムラインの出力ルール

- 字幕の発言と、それに対応するライブチャットを**時刻順に並べて1つのブロック内に配置**する
- 発言は `🎙️` プレフィックス、チャットは `💬` プレフィックスで区別する
- チャットコメントは直前の発言との**文脈的なつながり**を考慮して配置する（明らかな次のトピックへの先行反応は次のブロックに移してよい）
- 集約コメントは `💬 多数「〇〇」（N件以上）` の形式で記載する
- コメントが集中した場面（ハプニング・ボケ・感動シーン）は**コメントを多めに残す**

### 2-7. 統合タイムラインの出力形式

```
## 📺 統合タイムライン

[MM:SS]
🎙️ 発言内容。発言内容続き。
💬 コメント内容
💬 コメント内容
💬 多数「集約されたコメント」（N件以上）

[MM:SS]
🎙️ 発言内容。
💬 コメント内容

[MM:SS]
🎙️ [音楽]
```

## セクション③：通常コメント

### 処理ルール

- **全件掲載**（明らかなスパム・重複のみ省略）
- 投稿日時・いいね数・本文をそのまま使用
- **ユーザー名（@〇〇）は削除**する

### 出力形式

```
## 📨 通常コメント

（👍 N）*YYYY-MM-DD HH:MM UTC*
> コメント本文

---
```

## 全体的な注意事項

### ログ優先原則
- **情報の省略を最小限にする**：ノイズ（重複・タグ・フォーマット上のゴミ）のみ除去し、実質的な内容はすべて残す
- **要約・言い換えをしない**：発言内容は字幕から再構築したテキストをそのまま使う。AIによる要約や言い換えは行わない
- **チャットコメントは密に残す**：V2（まとめ版）より多くのコメントを採用する

### 字幕の品質について
- 自動認識エラーは辞書で補正した上で残す。補正できないものは `[※要確認:原文「〇〇」]` と注記する
- 認識が完全に破綻している箇所でも**削除せず**保持する
- 字幕に登場する固有名詞・ハッシュタグ・URLはそのまま保持する

### ユーザー名について
- ライブチャットおよび通常コメントの**投稿者名は全削除**
- 字幕（発言録）に記録された名前（配信者が読み上げた名前）は**保持**

### 読みやすさ
- タイムスタンプブロックごとに空行を入れる
- 出力言語は日本語（コメントが他言語の場合はそのまま残す）
- GitHub上でのMarkdownレンダリングを意識した書き方にする

---

## 処理フロー（内部参考）

1. `_info.txt` を解析し、動画情報ヘッダーを生成する
2. SRTを解析し、ノイズ除去→辞書補正→発話単位での再接続→時刻付き発言リストを生成する
3. ライブチャットを解析し、ノイズ除去→ユーザー名削除→集約処理→時刻付きコメントリストを生成する
4. 発言リストとコメントリストを**時刻順にマージ**し、統合タイムラインを構築する
5. 通常コメントを全件整形する（ユーザー名削除）
6. 全体をマークダウン形式で出力する
