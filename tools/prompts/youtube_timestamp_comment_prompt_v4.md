# YouTube配信ログからコメント用タイムスタンプ生成プロンプト v4

## このプロンプトの目的

YouTube Log Prompt で生成された統合ログファイル（`.compiled.md`）から、YouTube コメント欄に貼付可能なタイムスタンプ付きシーン説明を抽出し、複数のコメントに分割して出力する。

単なる事務的な事象の羅列（議事録）ではなく、配信の熱量や文脈（Vibe）が伝わる魅力的な目次を作成する。特に**「要約」ではなく「予告（Teaser）」**として機能させ、視聴者が「何があったのか」気になって思わずタイムスタンプをクリックしたくなるように誘導することを最大の目的とする。

---

## 入力と出力

**入力**：
- `.compiled.md`：統合ログファイル（SRT発言 + ライブチャット統合版）

**出力**：
- YouTube コメント欄用の複数コメント（各4000文字程度以下に分割）
- 形式：`[MM:SS] または [HH:MM:SS]` 形式のタイムスタンプ + シーン説明

---

## タイムスタンプ選別基準

統合ログに含まれるすべてのタイムスタンプから、以下のいずれかに該当するものを抽出する。

| カテゴリ | 具体的なシグナル | 優先度 |
|---------|----------------|--------|
| **ハプニング・事件** | 予期しない音声・エラー・バグ等（ポン酢音など） | 高 |
| **感動シーン** | チャットに「泣いた」「エモ」「いい話」など感動コメントが集中 | 高 |
| **笑いどころ・ネタ** | チャットに「草」「ｗ」「面白い」など笑いコメントが集中 | 中 |
| **マシュマロ読み** | マシュマロの開始地点＆内容（特に感動系・珍しい内容） | 中 |
| **キャラネタ・定番表現** | しずくの「勝ちます」「友達総会」など定番キャラネタ | 中 |
| **話題の転換点** | 配信の大きな構成転換（オープニング→マロ読み→エンディング等） | 低 |

## シーン説明の作成ルール

### ルール1：「要約」ではなく「予告（チラ見せ）」にする【最重要】
- オチ、オモシロ発言の答え、ハプニングの具体的な結末など、**「それを読んで満足してしまう（＝動画を見なくてよくなる）内容」は絶対に書かない。**
- 視聴者の「好奇心の空白（Curiosity Gap）」を刺激し、思わずクリックして答え合わせをしたくなるような寸止め表現にする。
- 良い例：`[28:30] 〇〇の話題でまさかの爆弾発言` / `[42:10] 〇〇バグ発生でチャット欄が阿鼻叫喚`
- 悪い例：`[28:30] 〇〇は嫌いだと発言し爆笑` / `[42:10] 画面が真っ暗になるバグ発生`（オチ・結果を書き切っているためNG）

### ルール1補足：「謎かけ→即答え」の連続ネタバレを防ぐ【最重要】
単体では問題のない記述でも、**前後のタイムスタンプの組み合わせで謎の答えが明かされてしまう**ケースに注意する。

- **「謎かけ」タイムスタンプの直後に「答え」タイムスタンプを置かない。**
  - NG例：`[33:56] チャット欄の予想を外す選曲` → `[34:09] 「〇〇」スタート`（直後に曲名で答えを出している）
  - OK例：`[33:56] チャット欄の予想を外す選曲` → `[34:09] しずくの声で聴くとまた色が変わる、しっとり系の一曲`（曲名を出さず別角度で切り取る）
- 「〇〇スタート」のような開始タイムスタンプは、直前に「どの曲？」という謎かけがある場合は特に危険。**曲名・オチを含まない別の切り口**（歌唱中の反応・ハプニング・特定フレーズの雰囲気など）に差し替えること。
- **連想ゲームの中身・発言の核心・ハプニングの顛末を列挙する説明はNG。** 中身の列挙自体がオチになる。
  - NG例：`[21:01] 鳥→焼き鳥→ビール→枝豆、謎の連想ゲーム`（連想の中身がオチそのもの）
  - OK例：`[21:01] 鳥の詩の余韻から突然始まった、行き先不明の連想ゲーム`

---

### ルール2：熱心なリスナー目線のトーン
- 事務的な議事録（例「〇〇について議論」）ではなく、配信を楽しんでいる熱心なファンの目線でテキストを書くこと。
- 「草」「エモい」「沼る」「限界化」など、ネット・配信カルチャーの語彙を自然に使って、文脈（Vibe）を反映させる。

### ルール3：適切なトピックの粒度を保つ
- 「雑談」などの大雑把すぎる見出しや、2〜3分単位で些細な出来事を並べる細かすぎる見出しはノイズになるため避ける。
- ひとつの話題の明確な区切りや、印象的なハイライトごとに抽出する。

### ルール4：簡潔さを優先
- 1つのシーン説明は **1〜2行**（最大30文字程度）に収める。
- YouTubeのUI（スマートフォン等）で見やすい長さを維持する。

### ルール5：チャット反応を活用
- そのシーンへの視聴者反応（チャットコメント）から意味を読み取り、説明に反映させる。
- 例：SRTが「吹いてあげます」のみでも、チャット反応が「ポン酢音実装」なら、説明は `突然のポン酢音（涙をなめる音）に湧くチャット欄` のようにする。

### ルール6：重要な人物名は記載
- 複数人が登場する配信の場合、誰のアクション・発言か明記する。

## 出力形式

### 複数コメント時のヘッダーと基本形式

YouTube コメント欄は1コメント約4000文字が上限。また、長文コメントは「続きを読む」で省略されるため、**折りたたまれない冒頭部分に最も魅力的なハイライトと注意書きを配置する。**

以下の形式で出力すること：

```
【タイムスタンプ集】① オープニング～マロ読み開始
🌟ハイライト：[MM:SS] （このパートで一番面白かった/気になりそうな箇所のタイムスタンプと寸止め説明）
※これは自動文字起こしからAIによって生成したタイムスタンプのため、ネタバレや誤り、時間のズレを含む可能性があります。

[MM:SS] シーン説明（簡潔な1〜2行）[MM:SS] シーン説明
…

---

【タイムスタンプ集】② マシュマロ① ～ マシュマロ③
🌟ハイライト：[MM:SS] （このパートの目玉シーン）[MM:SS] シーン説明
…

---

【タイムスタンプ集】③ マシュマロ④ ～ エンディング
🌟ハイライト：[MM:SS] （このパートの目玉シーン）

[MM:SS] シーン説明
…
```
※注意書きは最初のコメント（①）の冒頭にのみ記載する。

### 複数コメント分割ルール
- **1つのコメント = 大体20〜30個のタイムスタンプ＋説明**（文字数で4000文字以下）
- 分割地点は**配信の構成上の区切り目**を優先（オープニング→マロ読み開始など）
- 分割が不可避な場合は空行を入れて視認性を保つ

---

## 優先度に基づくフィルタリング

### 配信時間別のガイドライン

大雑把すぎる見出しや細かすぎる見出しを防ぐため、以下の目安に収まるように厳選すること。

| 配信時間 | タイムスタンプ数の目安 |
|---------|----------------------|
| 30分以下 | 10〜15個 |
| 30〜60分 | 20〜30個 |
| 60分以上 | 30〜50個 |

## 禁止事項

- [ ] オチ、結末、発言の核心など、動画を見る必要がなくなる「ネタバレ・完全な要約」を記載しない（チラ見せに徹する）
- [ ] 「どの曲？」「何が起きた？」と謎かけた直後のタイムスタンプで、その答えを書く「連続ネタバレ」をしない
-[ ] ログ（SRTとチャット）に存在しない事象や発言を勝手に推測・捏造（ハルシネーション）しない
- [ ] SRTの発言をそのまま丸写しにしない
- [ ] 「雑談」「ゲーム開始」など、中身が想像できない大雑把な見出しを作らない
- [ ] 個人特定につながるコメント内容は記載しない（マシュマロ投稿者の個人情報など）
- [ ] チャットのユーザー名を記載しない
- [ ] 説明を3行以上にしない（簡潔さを優先）
- [ ] YouTubeのリンクやスパムプラグを紛れ込ませない

---

## 出力品質のチェックリスト

- [ ] シーン説明が動画のオチを明かしておらず、「好奇心の空白」を刺激する予告（Teaser）になっているか
- [ ] 「謎かけ」タイムスタンプの直後に、その答えになるタイムスタンプを置いていないか（連続ネタバレチェック）
- [ ] 冒頭のハイライト設定など、YouTubeの「続きを読む」UIを考慮した配置になっているか
- [ ] 注意書きが指定通り「※これは自動文字起こしからAIによって生成した〜時間のズレを含む可能性があります。」となっているか
- [ ] 配信カルチャーに沿ったリスナー目線のトーン（Vibe）が反映されているか
- [ ] 「大雑把すぎる」「細かすぎる」タイムスタンプになっていないか
- [ ] ハプニング・感動シーンなどの重要なハイライトが含まれているか
- [ ] すべてのタイムスタンプが `[MM:SS]` または `[HH:MM:SS]` 形式か

---

## 処理フロー（内部参考）

1. `.compiled.md` を読み込む
2. 全タイムスタンプを抽出 → 発言とチャット反応をクロスチェック
3. トピックの粒度を調整し、ひとつの話題や事象ごとにまとめる
4. 各タイムスタンプに対して「予告（チラ見せ）」となるシーン説明を作成（リスナー目線・Vibe重視）
5. 配信時間に応じたタイムスタンプ数に厳選調整
6. 配信構成に基づいて複数コメントに分割（不要な場合は1コメント）
7. 各パートの「ハイライト」を選定し、ヘッダーに配置
8. 指定された注意書きを付与して出力
9. チェックリストで品質確認（特に「謎かけ→即答え」の連続ネタバレがないか、隣接タイムスタンプを前後で読み直して確認する）
