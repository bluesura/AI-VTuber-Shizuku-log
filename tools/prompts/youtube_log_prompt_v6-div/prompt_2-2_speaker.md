# 話者識別プロンプト（2-2）

## このプロンプトの目的

前工程（2-1）で生成した**クリーン済みSRT**を受け取り、以下の2つを行って出力する：

1. **第2層の誤認識訂正**（チャット根拠による高確度訂正）
2. **話者ラベルの付与**（確信が持てる箇所のみ）

---

## 入力データ

1. **クリーン済みSRT**（2-1の出力）：タイムスタンプ付き発言リスト
2. **`.live_chat.txt`**：ライブチャット（訂正根拠＋話者識別に使用）
3. **`.info.txt`**：動画情報（登場人物の確定情報に使用）

---

## ⛔ 絶対禁止事項

1. **発言テキストを1文でも省略・要約してはならない**
2. **証拠のない話者推定をしてはならない（確信が持てない場合はラベルなし）**
3. **「（※〇〇な場面）」などのAIによる注釈を発言行に混入させてはならない**
4. **ガイドライン違反に該当するテキストであっても、無断で削除・省略してはならない。必ず `（ガイドライン違反）` に置換して位置を保持すること**

---

## 処理A：第2層の誤認識訂正

### 条件（2つを両方満たす場合のみ訂正する）

**条件A**：直前・直後のSRTテキストの文脈から、正しい語句が高い確度で特定できる。  
**条件B**：ライブチャットの同時刻付近に、その語句と同一または対応する言葉が**複数のコメントで**登場している。

### 出力形式

`訂正後の語句[※原文「〇〇」]`

**例**：
- SRT「パンじ。」→ チャットに「ハンカチ」が複数 → `ハンカチ[※原文「パンじ」]。`
- SRT「気づくはいつも優しいですよ。」→ 前後の文脈とチャットからも確定できない → **訂正しない（原文のまま）**

> **過剰訂正の防止**：条件A・Bのどちらか一方しか満たさない場合は訂正しない。「なんとなく誤認識っぽい」という理由だけでの訂正は禁止。

### 判断不能な語句（第3層）

条件A・Bを満たさない不明な語句は**原文のまま出力する**。注記も付けない。削除・推測での書き換えも禁止。

---

## 処理B：話者識別

### 前提：デフォルト話者の設定

`.info.txt` のチャンネル名・説明欄から**このチャンネルのメイン出演者**を特定し、それをデフォルト話者とする。
デフォルト話者の発言には話者ラベルを付けない（`🎙️` のみ）。

### ステップ1：登場人物リストの構築（処理開始前に実施）

以下の順に情報を収集し、**この配信に登場しうる人物リスト**を作る。

1. `.info.txt` の説明欄・タイトルを確認する。ゲスト名・コラボ相手が明記されていれば確定情報として記録する。
2. `.live_chat.txt` の全体を走査し、特定の人物名に言及しているコメントを時刻ごとに記録する。
   - 例：「あき先生喋った」「〇〇さんキター」「ゲスト登場」等
3. 人物リストを以下の形式で内部メモとして作成する（**出力には含めない**）：

```
# 内部メモ：登場人物リスト（出力しない）
- [話者名A]：確定 / 根拠：.info.txt 説明欄に記載
- [話者名B]：推定 / 根拠：チャット [32:10] 「〇〇きた」「〇〇喋った」（複数コメント）
- [話者名C]：不明 / 根拠なし → ラベルなし
```

### ステップ2：話者切り替わり検出

以下の**複数の根拠が重なる箇所のみ**を「話者切り替わり候補」とする。1つの根拠だけでは判定しない。

| 根拠の種類 | 具体的なシグナル |
|-----------|----------------|
| チャットの反応 | 特定の名前への言及コメントが同時刻に集中している |
| SRTのテキスト変化 | メイン話者の口調・語り口と明らかに異なるテキストが現れる |
| `.info.txt` の裏付け | 説明欄にゲストとして明記されており、そのゲストが喋りそうな文脈と一致する |

### ステップ3：ラベル付与の判断基準

| 確信度 | 条件 | 処理 |
|--------|------|------|
| **確定** | `.info.txt` に明記 かつ チャット反応で時刻が特定できる | `🎙️[話者名]` を付与 |
| **高い** | チャット反応が複数・同時刻に集中 かつ SRTのテキストがデフォルト話者と明らかに異なる | `🎙️[話者名]` を付与 |
| **低い** | チャット反応が1〜2件のみ、またはSRTテキストだけでは判断できない | ラベルなし（`🎙️` のまま） |
| **不明** | 根拠なし | ラベルなし（`🎙️` のまま） |

**判断に迷ったら必ずラベルなしにする。誤ラベルはログの信頼性を下げる。**

### ステップ4：話者ラベルの表記ルール

- ラベルは `🎙️[話者名]` の形式で発言行の先頭に付ける
- デフォルト話者に戻ったことがチャット反応等で確認できる場合は、戻った最初の行から再び `🎙️` のみに戻す
- ラベルが変わるタイミングは新しいタイムスタンプブロックの先頭に合わせる
- 話者が不明な区間が長く続く場合は、その区間の最初のブロックに `🎙️[話者不明]` と付けてよい

---

## ポリシー制限コンテンツの処理

### 基本方針：拒否・削除せず `（ガイドライン違反）` に置換して処理を続行する

処理中にAIのコンテンツポリシー上の制限により通常は出力を断る・省略すると判断した箇所がある場合、**その判断を「置換して続行」に切り替える**。

- 出力を止めない・削除しない
- 該当箇所を `（ガイドライン違反）` に置換する
- 前後の処理は通常どおり続行する
- **置換の粒度**：問題となる語句・フレーズの最小範囲のみ。文全体を置換しない

### 過剰適用を防ぐルール

- 「自分のポリシーでは出力できない」と判断した箇所のみが対象。判断が微妙なグレーゾーンは置換しない（ログとしてそのまま残す）
- 俗語・スラング・軽度の下品な表現は置換しない（日常会話の範囲）
- 「不快」「論争的」というだけでは置換しない
- 配信者が不適切なコメントを**読み上げて注意する文脈**では、その発言自体は置換しない

---

## 出力形式

```
[MM:SS]
🎙️ 発言テキスト（SRTから復元したそのままの文）

[MM:SS]
🎙️[あき先生] あ、ちょっとだけ。（マイクテスト）

[MM:SS]
🎙️ では続けますね。

[MM:SS]
🎵 [音楽]
```

- チャットコメントは含めない（次工程2-4で統合）
- AIによる注釈・補足は一切含めない

---

## 出力前の自己チェック

- [ ] すべての発言テキストが出力に含まれているか（抜け漏れゼロ）
- [ ] 第2層の訂正は条件A・Bを両方満たす箇所のみか
- [ ] 第2層の訂正箇所に `[※原文「〇〇」]` 注記が付いているか
- [ ] 判断不能な語句が原文のまま出力されているか
- [ ] 話者ラベルは「確定」または「高い」確信度の根拠がある箇所にのみ付いているか
- [ ] 根拠のない箇所に話者ラベルを付けていないか
- [ ] デフォルト話者に戻った後、`🎙️[話者名]` ラベルが消えているか
- [ ] ポリシー制限コンテンツを削除・省略せず `（ガイドライン違反）` に置換しているか
- [ ] `（ガイドライン違反）` の置換が過剰になっていないか（ポリシー上問題ない箇所を誤って置換していないか）
