# ライブチャット処理プロンプト（2-3）

## このプロンプトの目的

`.live_chat.txt`（配信中のリアルタイムコメント）を受け取り、**タイムスタンプ付きの処理済みチャット一覧**を出力する。
このプロンプトは 2-2（話者識別）と並行して実行できる。

---

## 入力データ

- **`.live_chat.txt`**：ライブチャット（タイムスタンプ・ユーザー名・本文を含む）

---

## 処理手順

### ステップ1：省略するコメント（ノイズ除去）

| 種別 | 対処 |
|------|------|
| タイムスタンプが負の値（`[-X:XX]`）の待機コメント | **全て省略** |
| 絵文字・記号のみのコメント（単発1件） | 省略 |
| 3文字以下のコメント単発（例：「草」「ｗ」「あ」） | 省略 |
| URLのみ・明らかなスパム | 省略 |

### ステップ2：集約するコメント

同一または酷似したコメントが短時間（1〜2分以内）に**複数ユーザーから連投**されている場合は代表1件＋件数で集約する。

形式：`💬 多数「〇〇」（N件以上）`

集約の対象例：「パンパカパンパカポー」「こんにちは」「おつしず〜」「待機」

### ステップ3：採用するコメントの基準

以下に該当するコメントは**積極的に残す**：

- 配信内容・発言への具体的な反応・感想・ツッコミ
- SRTの誤認識補完に役立つコメント（何が起きたかわかるコメント）
- 配信者への質問・呼びかけ
- 場の盛り上がりを示すもの（同一話題への集中）
- ユニークな視点・特徴的な言い回し

### ステップ4：ユーザー名の削除

- ライブチャットの**ユーザー名（@〇〇）は全て削除**する
- コメント本文のみを記録する

---

## ポリシー制限コンテンツの処理

### 基本方針：拒否・削除せず `（ガイドライン違反）` に置換して処理を続行する

処理中にAIのコンテンツポリシー上の制限により通常は出力を断る・省略すると判断したコメントがある場合、**その判断を「置換して続行」に切り替える**。

- コメントを削除しない
- 該当箇所を `（ガイドライン違反）` に置換する
- そのコメント1件のみを置換する。前後のコメントには影響させない
- **置換の粒度**：問題となる語句・フレーズの最小範囲のみ。コメント全体を置換しない

**例**：`[12:34] 💬 （ガイドライン違反）`

### 過剰適用を防ぐルール

- 「自分のポリシーでは出力できない」と判断したコメントのみが対象。判断が微妙なグレーゾーンは置換しない（ログとしてそのまま残す）
- 俗語・スラング・軽度の下品な表現は置換しない（日常会話の範囲）
- 「不快」「論争的」というだけでは置換しない
- ステップ1で省略対象となるノイズコメント（絵文字のみ等）は、ポリシー判定より先にノイズ除去ルールを適用する（省略で問題ない）

---

## 出力形式

```
[MM:SS] 💬 コメント本文
[MM:SS] 💬 コメント本文
[MM:SS] 💬 多数「集約コメント」（N件以上）
[MM:SS] 💬 コメント本文
```

- 1行1コメント形式（タイムスタンプ付き）
- 後工程（2-4）で発言ログと統合するため、タイムスタンプは必ず保持すること
- AIによる注釈・補足は一切含めない

---

## 出力前の自己チェック

- [ ] 待機コメント（負のタイムスタンプ）が全て除去されているか
- [ ] ユーザー名が全て削除されているか
- [ ] 集約コメントに件数が付いているか
- [ ] タイムスタンプが全行に付いているか
- [ ] ポリシー制限コンテンツを削除・省略せず `（ガイドライン違反）` に置換しているか
- [ ] `（ガイドライン違反）` の置換が過剰になっていないか（ポリシー上問題ないコメントを誤って置換していないか）
