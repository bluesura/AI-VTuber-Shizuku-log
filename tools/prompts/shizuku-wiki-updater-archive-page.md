# AI VTuber しずく — Wiki「動画・配信」ページ更新スキル

## 概要

AI VTuber「しずく」の公式YouTubeチャンネルと既存のFandom wikiソースを照合し、
未掲載の動画・配信だけを追加した上で「動画・配信」ページの更新済みwikiソースを生成する。

---

## ステップ0: wikiソースの添付確認（必須・最初に行う）

**作業開始前に、必ずユーザーが現在のwikiソースを添付していることを確認する。**

- 添付されていない場合は作業を進めず、以下のメッセージを返す：

  > 作業を開始する前に、Fandom wikiの「動画・配信」ページの**現在のwikiソース（全文）**を貼り付けてください。
  > ページ編集画面を開き、ソースエディタに切り替えた上でテキスト全体をコピーして貼り付けてください。

- ソースが確認できたら「wikiソースを確認しました。YouTubeを確認して差分を調べます。」と一言伝えてから次のステップへ進む。

---

## ステップ1: wikiソースから既存エントリを抽出

添付されたwikiソースを解析し、現在すでに掲載されている全エントリのURLリスト（`watch?v=VIDEO_ID`）を抽出する。

- テーブル内の `[https://www.youtube.com/watch?v=... タイトル]` の形式からIDを取得する
- 「2026年アーカイブ」「2023年アーカイブ」「通常動画」「ショート」すべてのセクションを対象にする
- 抽出したIDはステップ2で「すでに掲載済み」と判断するために使う

---

## ステップ2: YouTubeから未掲載エントリのみ取得

以下のURLを `web_fetch` で取得し、**ステップ1で抽出したIDに含まれないもの（未掲載）だけ**を収集する。

### 取得するページ

1. **ライブ配信一覧（アーカイブ）**
   ```
   https://www.youtube.com/@aivtuber_shizuku/streams
   ```

2. **通常動画一覧**
   ```
   https://www.youtube.com/@aivtuber_shizuku/videos
   ```

3. **ショート一覧**
   ```
   https://www.youtube.com/@aivtuber_shizuku/shorts
   ```

### 日付の取得方針（重要）

- YouTubeの「X日前」「X週間前」「X年前」などの **相対表記は使用禁止**。
- 相対表記しか取れない場合は、該当動画の個別ページを `web_fetch` して日付を確認する：
  ```
  https://www.youtube.com/watch?v={VIDEO_ID}
  ```
  説明欄・メタデータ・og:description 等から `YYYY-MM-DD` を特定する。
- それでも特定できない場合は、日付欄を **空欄のまま** にする。
  - 「ごろ」「不明」「推定」などの曖昧な表記は一切使わない。
  - 空欄になった動画はステップ4の「手動確認リスト」に記載する。
- 時刻が判明している場合のみ `HH:MM` を付記する。判明しない場合は日付のみ `YYYY-MM-DD`。

---

## ステップ3: 各未掲載エントリの情報を整理

未掲載の動画・配信について以下の情報を収集する：

| 項目 | 内容 |
|------|------|
| 公開日時（JST） | `YYYY-MM-DD` または `YYYY-MM-DD HH:MM`。不明なら空欄 |
| タグ | 下記タグ分類表を参照。判断できない場合は空欄 |
| 時間 | `H:MM:SS` または `MM:SS` 形式。不明なら空欄 |
| タイトル | YouTubeのタイトルをそのままコピー |
| URL | `https://www.youtube.com/watch?v=VIDEO_ID` |

### タグ分類表

タイトルから判断できるものだけタグを付ける。判断できない場合は空欄でよい。

| タグ | 判断基準 |
|------|---------|
| 雑談 | 「雑談」「おしゃべり」「お話し」を含む |
| ASMR | 「ASMR」「バイノーラル」「朗読ASMR」を含む |
| 歌枠 | 「歌枠」「歌配信」「歌います」を含む（ライブ配信）|
| 歌ってみた | 「歌ってみた」を含む（通常動画） |
| ゲーム | 「マインクラフト」「Minecraft」「APEX」「ゲーム」などを含む |
| 企画 | 「裁判」「AI裁判」「大喜利」「クイズ」「選考会」「マシュマロ」「エイプリルフール」「七夕」「ハロウィン」「逆転」「指導」などを含む |
| 記念 | 「初配信」「記念」「突破」「ただいま」を含む |
| 耐久 | 「耐久」を含む |
| おでかけ | 「おでかけ」「水族館」「潜入」など外出系を含む |
| 紹介 | 「初見向け」「分かる」「紹介」など説明系を含む |
| レポート | 「GDC」「展示会」「レポート」など外部イベントを含む |
| まとめ | 「まとめ」を含む |

### 配信種別の分類

- **ライブ配信アーカイブ** → 該当年の `== 20XX年アーカイブ ==` に追加
- **通常動画**（ライブ配信でない動画） → `== 通常動画 ==` に追加
- **ショート**（60秒以下・縦型） → `== ショート ==` に追加（セクションがなければ新設）

## ステップ4: 手動確認リストの出力

日付が特定できなかった未掲載エントリを、wikiソースとは**別に**まとめて出力する。

形式：

```
【手動確認が必要な動画・配信】
以下は投稿日を自動取得できませんでした。各URLを直接確認して日付を補完してください。

1. タイトル: {タイトル}
   URL: https://www.youtube.com/watch?v={VIDEO_ID}
   備考: {理由、例「個別ページでも日付の記述が見つからなかった」}

2. ...
```

手動確認リストがない（すべての日付を特定できた）場合は「すべての日付を特定できました」と一言添える。

---

## ステップ5: wikiソースの更新と出力

添付されたwikiソースをベースに、未掲載エントリを追加し、以下の項目を更新した上でwikiソース全文を出力する。

### 更新する箇所

1. **各年のアーカイブテーブル** — 未掲載エントリを新しい日付順で適切な位置に挿入
2. **通常動画テーブル** — 未掲載の通常動画を挿入
3. **ショートテーブル** — 該当があれば挿入。セクション未設の場合は `== 通常動画 ==` の後に新設
4. **配信状況欄** — ライブ配信の本数・年別内訳を再集計して更新

### フォーマットルール

- テーブルは **新しい日付順**（降順）
- 年をまたぐ場合は年ごとにセクションを分ける
- 日付・タグ・時間が空欄の行は該当セルを空白のまま残す
- URLは短縮しない
- `class="wikitable sortable"` を必ず維持する
- **既存エントリの内容（タイトル・URL・タグ・時間・日付）は変更しない**

### 出力形式

まず変更サマリーを簡潔に伝える：

```
【変更サマリー】
- 新規追加: X本（内訳: ライブ配信 X本、通常動画 X本）
- 配信状況: XX本 → XX本（2023年 48本 / 2026年 XX本）
```

次に、更新済みwikiソース全文をコードブロックで出力する：

` ```wiki `
（wikiソース全文）
` ``` `

---

## 注意事項

- **ステップ0を必ず最初に実行する**。wikiソースの添付なしに作業を開始しない。
- 既存エントリの内容は書き換えない。追加と本数更新のみ行う。
- 「ごろ」「不明」「推定」などの曖昧な日付表記は一切使わない。日付不明なら空欄、そして手動確認リストへ。
- YouTubeはJavaScriptレンダリングのため `web_fetch` で取得できる情報が限られる場合がある。取得できなかった動画は手動確認リストに回す。
- タイトルはYouTubeのものをそのままコピーし、改変しない。
- 出力はMediaWiki記法のみ。Markdown形式では出力しない。
