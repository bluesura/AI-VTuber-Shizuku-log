# SRTクリーニングプロンプト（2-1）

## このプロンプトの目的

`.srt.txt`（YouTube自動生成字幕）を受け取り、**クリーン済みSRT中間ファイル**を出力する。
次工程（2-2）で話者識別・第2層誤認識訂正を行うため、ここでは**テキスト内容を変えず、書式だけを整える**。

---

## ⛔ 絶対禁止事項

1. **発言テキストを1文でも省略してはならない**
2. **発言内容をAIが要約・言い換えしてはならない**
3. **「（※〇〇な場面）」などのAIによる注釈を発言行に混入させてはならない**

---

## 処理手順（厳密に順番どおりに実行すること）

### ステップ1：書式ノイズの除去

以下のものだけを除去する。**発言テキストは一切除去しない。**

- 行頭の連番（数字のみの行）
- `-->` を含むタイムスタンプ行（内容が空のブロック含む）
- `align:start position:0%` などのメタデータ
- `<00:00:00,000><c>テキスト</c>` 形式のインライン単語タイミングタグ（`<c>`～`</c>` 部分のみ除去し、テキスト本体は残す）
- 行頭が半角スペース1文字のみの空行
- 完全な空行

### ステップ2：重複行の除去

SRTでは前のブロックの末尾テキストが次のブロックの先頭行に**そのまま再掲**されるパターンがある。
この再掲行のみを除去し、**そのブロックで新たに追加されたテキスト断片だけを残す。**

具体例：
```
（ブロックA）ポン酢の涙を流すご主人様。雫が
（ブロックB先頭に再掲）ポン酢の涙を流すご主人様。雫が  ← 除去
（ブロックBの新規追加分）吹いてあげます。              ← 残す
```

### ステップ3：発言断片の結合

同一発話の断片を**句点（。）または明確な文の区切り**を基準に1行に繋ぐ。

- 結合は形式的な連結のみ。**テキスト内容を変えてはならない。**
- `[音楽]` がテキスト途中に埋め込まれている場合（例：`ご主人様を[音楽]大切に`）は、**`[音楽]`を除去してテキストを繋ぎ、`ご主人様を大切に`** とする。ただし**その発言の文意が[音楽]なしで通るときのみ**この処理をする。文意が不明な場合は `[音楽]` を残す。
- 音楽のみのブロック（発言テキストがなく `[音楽]` しかないブロック）は **`🎵 [音楽]`** として独立行で残す。

### ステップ4：第1層の誤認識訂正のみ適用

辞書に載っている語を**注記なしで**そのまま置換する。

| SRT上の誤認識表記 | 正しい表記 | 備考 |
|----------------|-----------|------|
| 秋先生 / 脇先生 / 秋先生 / 明先生 / あ先生 | あき先生 | 開発者名 |
| しず子 / しず / 雫 | しずく | 配信者名 |
| 同説 | 同接 | 同時接続数の略 |

> ⚠️ 第2層（チャット根拠による訂正）は**このプロンプトでは行わない**。次工程（2-2）で実施する。

### ステップ5：タイムスタンプの割り当て

各発言行に、そのブロックの**開始タイムスタンプ**を割り当てる。形式は `[MM:SS]` または `[HH:MM:SS]`。

**タイムスタンプ密度の方針**（ログ目的のため密に打つ）：
- 発話が途切れる区切り（無音・`🎵 [音楽]` 区間の後）ごとに打つ
- 同一発話が続く場合は**30〜60秒に1回**を目安に打つ
- 話題の明確な切り替わり（マシュマロ読み開始など）は必ず打つ

---

## 出力形式

```
[MM:SS]
発言テキスト（クリーン済み・1文1行）
発言テキスト（続き）

[MM:SS]
🎵 [音楽]

[MM:SS]
発言テキスト
```

- 話者ラベル（🎙️）は付けない（次工程で付与）
- チャットコメントは含めない
- AIによる注釈・補足は一切含めない

---

## 出力前の自己チェック

- [ ] すべての発言テキストが出力に含まれているか（抜け漏れゼロ）
- [ ] AIが発言内容を要約・言い換えていないか
- [ ] 第1層（辞書）の置換が正しく適用されているか
- [ ] 音楽区間が `🎵 [音楽]` として正しく残っているか
- [ ] タイムスタンプが適切な密度で打たれているか
