# 統合・最終出力プロンプト（2-4）

## このプロンプトの目的

前工程の出力を統合し、**GitHubで公開するアーカイブログ**をマークダウン形式で生成する。

---

## 入力データ

1. **話者ラベル付きSRT**（2-2の出力）：タイムスタンプ＋🎙️ラベル付き発言一覧
2. **処理済みチャット**（2-3の出力）：タイムスタンプ付きコメント一覧
3. **`.info.txt`**：動画情報（ヘッダー生成に使用）
4. **`.comments.txt`**：配信後の通常コメント

---

## 処理A：動画情報ヘッダーの生成（セクション①）

`.info.txt` の内容をもとに以下の形式で出力する。説明欄は全文そのまま記載する。

```markdown
# 【動画タイトル】

> **URL**: https://www.youtube.com/watch?v=XXXXXXX
> **動画ID**: XXXXXXX
> **配信日**: YYYY-MM-DD
> **チャンネル**: チャンネル名
> **説明欄**:
> （説明欄の内容をそのまま記載）
```

---

## 処理B：統合タイムラインの構築（セクション②）

2-2の発言リストと2-3のコメントリストを**時刻順にマージ**して統合タイムラインを生成する。

### 配置ルール

- チャットコメントは直前の発言との文脈的つながりを優先して配置する
- 明らかに次の発言への先行反応の場合は次のブロックに移してよい
- コメントが集中したハプニング・盛り上がり場面は**多めに残す**
- タイムスタンプはSRTの区切りに合わせる（コメントのみの時刻でブロックを新設しない）

### 出力形式

```markdown
## 📺 統合タイムライン

[MM:SS]
🎙️ 発言テキスト（SRTから復元したそのままの文）
💬 チャットコメント
💬 チャットコメント
💬 多数「集約コメント」（N件以上）

[MM:SS]
🎵 [音楽]
💬 チャットコメント（音楽区間中のコメント）

[MM:SS]
🎙️[ゲスト名] 発言テキスト（話者ラベルあり）
💬 チャットコメント

[MM:SS]
🎙️ 発言テキスト（デフォルト話者に戻った）
💬 チャットコメント
```

---

## 処理C：通常コメントの整形（セクション③）

**全件掲載**する（明らかなスパム・完全重複のみ省略）。
ユーザー名（@〇〇）は削除する。投稿日時・いいね数・本文をそのまま使用する。

```markdown
## 📨 通常コメント

（👍 N）*YYYY-MM-DD HH:MM UTC*
> コメント本文

---
```

---

## 全体の出力構成

```markdown
# 【動画タイトル】

> **URL**: ...
> **動画ID**: ...
> **配信日**: YYYY-MM-DD
> **チャンネル**: ...
> **説明欄**:
> ...

---

## 📺 統合タイムライン
...

---

## 📨 通常コメント
...
```

---

## 出力前の自己チェック

- [ ] すべての発言テキストが統合タイムラインに含まれているか（2-2の出力から抜け漏れゼロ）
- [ ] AIが発言内容を要約・言い換えていないか
- [ ] ユーザー名がチャット・コメントから削除されているか
- [ ] 音楽区間が `🎵 [音楽]` として正しく残っているか
- [ ] 話者ラベルが正しく引き継がれているか
- [ ] デフォルト話者に戻った後、`🎙️[話者名]` ラベルが消えているか
- [ ] 通常コメントが全件掲載されているか（スパム・完全重複のみ省略）
