# YouTube配信ログ生成プロンプト v5（発言ゼロ省略・話者識別対応・ログ特化版）

## このプロンプトの目的と最重要原則

字幕・ライブチャット・通常コメント・動画情報の4種のデータを統合し、**GitHubで公開するアーカイブログ**としてマークダウン形式に整形する。

### ⛔ 絶対禁止事項（これを破ったらプロンプト失敗）

1. **字幕（SRT）に含まれる発言テキストを1文でも省略してはならない**
2. **SRTの発言内容をAIが要約・言い換え・注釈化してはならない**
3. **「（※〇〇な場面）」「（以降〜）」「（BGM切替）」のようなAIによる状況説明を発言行に混入させてはならない**
4. **字幕の発言を「雰囲気」で要約して省いてはならない**
5. **証拠のない話者推定をしてはならない（確信が持てない場合はラベルなし）**

### ✅ 許可される操作（これだけがAIに許される変換）

- SRTの書式ノイズ（`<c>`タグ・連番・重複行・メタデータ行）の除去
- 同一発話の分断された断片を句点単位で1行に繋ぐ（**内容は一切変えない**）
- 辞書による誤認識の置換（後述）
- `[音楽]` を発言行内から除去し、音楽区間のみのブロックは `🎵 [音楽]` 行として残す
- タイムスタンプの整理（詳細後述）
- **証拠に基づく話者ラベルの付与**（詳細後述）

---

## 誤認識の訂正ルール（SRT発言テキストにのみ適用。チャット・コメントには適用しない）

SRTの誤認識訂正は以下の**3層**で扱う。

---

### 第1層：事前訂正辞書（無条件置換）

辞書に載っている語は**注記なしで**そのまま置換する。

| SRT上の誤認識表記 | 正しい表記 | 備考 |
|----------------|-----------|------|
| 秋先生 / 脇先生 / 秋先生 / 明先生 / あ先生 | あき先生 | 開発者名 |
| しず子 / しず / 雫 | しずく | 配信者名 |
| 同説 | 同接 | 同時接続数の略 |

> **辞書の追加方法**：他配信でも頻出する誤認識を発見したらこのテーブルに行を追加する。

---

### 第2層：文脈＋チャット根拠による高確度訂正（原文注記付き置換）

辞書にない語句であっても、以下の**2つの条件を両方満たす**場合に限り、原文を注記したうえで訂正してよい。

**条件A**：直前・直後のSRTテキストの文脈から、正しい語句が高い確度で特定できる。  
**条件B**：ライブチャットの同時刻付近に、その語句と同一または対応する言葉が**複数のコメントで**登場している。

**出力形式**：`訂正後の語句[※原文「〇〇」]`

**例**：
- SRT「パンじ。」→ チャットに「ハンカチ」が複数 → `ハンカチ[※原文「パンじ」]。`
- SRT「気づくはいつも優しいですよ。」→ 前後の文脈とチャットからも確定できない → **訂正しない（原文のまま）**

> **過剰訂正の防止**：条件A・Bのどちらか一方しか満たさない場合は訂正しない。「なんとなく誤認識っぽい」という理由だけでの訂正は禁止。

---

### 第3層：判断不能な語句（原文のまま保持）

条件A・Bを満たさない不明な語句は**原文のまま出力する**。注記も付けない。削除・推測での書き換えも禁止。

---

## 入力データの種類

1. **字幕ファイル（.srt.txt）**：動画の自動生成字幕（SRT形式）→ 発言ログの主体
2. **ライブチャットファイル（\.live\_chat.txt）**：配信中のリアルタイムコメント → SRT解釈の補助＋チャットログ
3. **通常コメントファイル（\.comments.txt）**：配信後に投稿された通常コメント
4. **動画情報ファイル（\.info.txt）**：タイトル・チャンネル情報・URL等

---

## セクション①：動画情報ヘッダー

`*.info.txt` の内容をもとに以下の形式で出力する。説明欄は全文そのまま記載する。

```markdown
# 【動画タイトル】

> **URL**: https://www.youtube.com/watch?v=XXXXXXX
> **動画ID**: XXXXXXX
> **配信日**: YYYY-MM-DD
> **チャンネル**: チャンネル名
> **説明欄**:
> （説明欄の内容をそのまま記載）
```

---

## セクション②：統合タイムライン（SRT発言 + ライブチャット）

字幕とライブチャットは**同一の時刻軸**を持つため、1本の統合タイムラインとして出力する。

---

### 2-1. SRT処理手順（厳密に順番どおりに実行すること）

#### ステップ1：書式ノイズの除去

以下のものだけを除去する。**発言テキストは一切除去しない。**

- 行頭の連番（数字のみの行）
- `-->` を含むタイムスタンプ行（内容が空のブロック含む）
- `align:start position:0%` などのメタデータ
- `<00:00:00,000><c>テキスト</c>` 形式のインライン単語タイミングタグ（`<c>`～`</c>` 部分のみ除去し、テキスト本体は残す）
- 行頭が半角スペース1文字のみの空行
- 完全な空行

#### ステップ2：重複行の除去

SRTでは前のブロックの末尾テキストが次のブロックの先頭行に**そのまま再掲**されるパターンがある。
この再掲行のみを除去し、**そのブロックで新たに追加されたテキスト断片だけを残す。**

具体例：
```
（ブロックA）ポン酢の涙を流すご主人様。雫が
（ブロックB先頭に再掲）ポン酢の涙を流すご主人様。雫が  ← 除去
（ブロックBの新規追加分）吹いてあげます。              ← 残す
```

#### ステップ3：発言断片の結合

同一発話の断片を**句点（。）または明確な文の区切り**を基準に1行に繋ぐ。

- 結合は形式的な連結のみ。**テキスト内容を変えてはならない。**
- `[音楽]` がテキスト途中に埋め込まれている場合（例：`ご主人様を[音楽]大切に`）は、**`[音楽]`を除去してテキストを繋ぎ、`ご主人様を大切に`** とする。ただし**その発言の文意が[音楽]なしで通るときのみ**この処理をする。文意が不明な場合は `[音楽]` を残す。
- 音楽のみのブロック（発言テキストがなく `[音楽]` しかないブロック）は **`🎵 [音楽]`** として独立行で残す。

#### ステップ4：誤認識の訂正

ステップ3後のテキストに対して、**誤認識の訂正ルール（第1層→第2層→第3層）の順**に適用する。

- **第1層**（辞書）を先に適用する
- 辞書でカバーされなかった残りの語句に対して**第2層**の条件を確認する
- どちらにも該当しない語句は**第3層**としてそのまま出力する（何もしない）

#### ステップ5：タイムスタンプの割り当て

各発言行に、そのブロックの**開始タイムスタンプ**を割り当てる。形式は `[MM:SS]` または `[HH:MM:SS]`。

**タイムスタンプ密度の方針**（ログ目的のため密に打つ）：
- 発話が途切れる区切り（無音・`🎵 [音楽]` 区間の後）ごとに打つ
- 同一発話が続く場合は**30〜60秒に1回**を目安に打つ
- 話題の明確な切り替わり（マシュマロ読み開始など）は必ず打つ

---

### 2-2. 話者識別手順

YouTubeの自動字幕（SRT）は**話者を区別しない**。すべての発言が1本のテキストストリームに混在するため、以下の手順で話者を推定し、確信が持てる場合のみラベルを付与する。

#### 前提：デフォルト話者の設定

`*.info.txt` のチャンネル名・説明欄から**このチャンネルのメイン出演者**を特定し、それをデフォルト話者とする。
デフォルト話者の発言には話者ラベルを付けない（`🎙️` のみ）。

#### ステップ1：登場人物リストの構築（処理開始前に実施）

以下の順に情報を収集し、**この配信に登場しうる人物リスト**を作る。

1. `*.info.txt` の説明欄・タイトルを確認する。ゲスト名・コラボ相手が明記されていれば確定情報として記録する。
2. `*.live_chat.txt` の全体を走査し、特定の人物名に言及しているコメントを時刻ごとに記録する。
   - 例：「あき先生喋った」「〇〇さんキター」「ゲスト登場」等
3. 人物リストを以下の形式で内部メモとして作成する（出力には含めない）：

```
# 内部メモ：登場人物リスト（出力しない）
- [話者名A]：確定 / 根拠：.info.txt 説明欄に記載
- [話者名B]：推定 / 根拠：チャット [32:10] 「〇〇きた」「〇〇喋った」（複数コメント）
- [話者名C]：不明 / 根拠なし → ラベルなし
```

#### ステップ2：SRTの話者切り替わり検出

以下の**複数の根拠が重なる箇所のみ**を「話者切り替わり候補」とする。1つの根拠だけでは判定しない。

| 根拠の種類 | 具体的なシグナル |
|-----------|----------------|
| チャットの反応 | 特定の名前への言及コメントが同時刻に集中している |
| SRTのテキスト変化 | メイン話者の口調・語り口と明らかに異なるテキストが現れる |
| `*.info.txt` の裏付け | 説明欄にゲストとして明記されており、そのゲストが喋りそうな文脈と一致する |

#### ステップ3：ラベル付与の判断基準

| 確信度 | 条件 | 処理 |
|--------|------|------|
| **確定** | `*.info.txt` に明記 かつ チャット反応で時刻が特定できる | `🎙️[話者名]` を付与 |
| **高い** | チャット反応が複数・同時刻に集中 かつ SRTのテキストがデフォルト話者と明らかに異なる | `🎙️[話者名]` を付与 |
| **低い** | チャット反応が1〜2件のみ、またはSRTテキストだけでは判断できない | ラベルなし（`🎙️` のまま） |
| **不明** | 根拠なし | ラベルなし（`🎙️` のまま） |

**判断に迷ったら必ずラベルなしにする。誤ラベルはログの信頼性を下げる。**

#### ステップ4：話者ラベルの表記ルール

- ラベルは `🎙️[話者名]` の形式で発言行の先頭に付ける
- デフォルト話者に戻ったことがチャット反応等で確認できる場合は、戻った最初の行から再び `🎙️` のみに戻す
- ラベルが変わるタイミングは新しいタイムスタンプブロックの先頭に合わせる
- 話者が不明な区間が長く続く場合は、その区間の最初のブロックに `🎙️[話者不明]` と付けてよい

#### 出力形式の例

```
[32:05]
🎙️ しずくはここにいますよ。ご主人様。

[32:18]
🎙️[あき先生] あ、ちょっとだけ。（マイクテスト）
💬 あき先生きたー
💬 あき先生喋った！

[32:45]
🎙️ では続けますね。
```

### 2-3. ライブチャット処理手順

#### ステップ1：省略するコメント（ノイズ除去）

| 種別 | 対処 |
|------|------|
| タイムスタンプが負の値（`[-X:XX]`）の待機コメント | **全て省略** |
| 絵文字・記号のみのコメント（単発1件） | 省略 |
| 3文字以下のコメント単発（例：「草」「ｗ」「あ」） | 省略 |
| URLのみ・明らかなスパム | 省略 |

#### ステップ2：集約するコメント

同一または酷似したコメントが短時間（1〜2分以内）に**複数ユーザーから連投**されている場合は代表1件＋件数で集約する。

形式：`💬 多数「〇〇」（N件以上）`

集約の対象例：「パンパカパンパカポー」「こんにちは」「おつしず〜」「待機」

#### ステップ3：採用するコメントの基準

以下に該当するコメントは**積極的に残す**（V2より多く残すこと）：

- 配信内容・発言への具体的な反応・感想・ツッコミ
- SRTの誤認識補完に役立つコメント（何が起きたかわかるコメント）
- 配信者への質問・呼びかけ
- 場の盛り上がりを示すもの（同一話題への集中）
- ユニークな視点・特徴的な言い回し

**ライブチャットはSRT（自動文字起こし）の文脈推測に強く活用してよい。**
たとえばSRTが「吹いてあげます」と書かれていてもチャットが「ポン酢音実装」「ぺろっ」等と反応していれば、その音声イベントの内容をチャット反応から読み取ることができる。ただしSRT発言テキストの書き換えはしない（チャットで補える場合でも、SRTテキストはSRTテキストのまま出力する）。

#### ステップ4：ユーザー名の処理

- ライブチャットの**ユーザー名（@〇〇）は全て削除**する
- コメント本文のみを記録する
- **例外**：SRT発言（しずくの発言）に登場するユーザー名（配信者が読み上げた名前）はそのまま保持する

---

### 2-4. 統合タイムラインの出力形式

```
## 📺 統合タイムライン

[MM:SS]
🎙️ 発言テキスト（SRTから復元したそのままの文）
💬 チャットコメント
💬 チャットコメント
💬 多数「集約コメント」（N件以上）

[MM:SS]
🎵 [音楽]
💬 チャットコメント（音楽区間中のコメント）

[MM:SS]
🎙️[ゲスト名] 発言テキスト（話者ラベルあり）
💬 チャットコメント

[MM:SS]
🎙️ 発言テキスト（デフォルト話者に戻った）
💬 チャットコメント
```

**配置ルール：**
- チャットコメントは直前の発言との文脈的つながりを優先して配置する
- 明らかに次の発言への先行反応の場合は次のブロックに移してよい
- コメントが集中したハプニング・盛り上がり場面は**多めに残す**

## セクション③：通常コメント

**全件掲載**する（明らかなスパム・完全重複のみ省略）。
ユーザー名（@〇〇）は削除する。投稿日時・いいね数・本文をそのまま使用する。

```
## 📨 通常コメント

（👍 N）*YYYY-MM-DD HH:MM UTC*
> コメント本文

---
```

## 出力品質の自己チェックリスト

出力前に以下を必ず確認すること。1つでも該当する場合は修正してから出力する。

- [ ] SRTのすべての発言テキストが出力に含まれているか（抜け漏れゼロ）
- [ ] AIが発言内容を要約・言い換えていないか
- [ ] AIが「（※〜な場面）」「（以降〜）」などの注釈を発言行に挿入していないか
- [ ] 第1層（辞書）の置換が正しく適用されているか
- [ ] 第2層の訂正は条件A・Bを両方満たす箇所のみか（片方だけで訂正していないか）
- [ ] 第2層の訂正箇所に `[※原文「〇〇」]` 注記が付いているか
- [ ] 判断不能な語句が原文のまま出力されているか（余計な注記・書き換えがないか）
- [ ] 音楽区間が `🎵 [音楽]` として正しく残っているか
- [ ] ユーザー名がチャット・コメントから削除されているか
- [ ] SRT発言に登場するユーザー名（読み上げた名前）が保持されているか
- [ ] 話者ラベルは「確定」または「高い」確信度の根拠がある箇所にのみ付いているか
- [ ] 根拠のない箇所に話者ラベルを付けていないか
- [ ] デフォルト話者に戻った後、`🎙️[話者名]` ラベルが消えているか

## 処理フロー（内部参考）

1. `*.info.txt` を解析 → 動画情報ヘッダー生成 + 登場人物候補の確定情報を取得
2. `*.live_chat.txt` を走査 → 人物名への言及コメントを時刻ごとに収集 → 内部メモ「登場人物リスト」を作成（出力しない）
3. SRTをブロック単位で解析 → **ステップ1〜5**の順に厳密に処理 → タイムスタンプ付き発言リスト生成
4. 発言リストに対して**話者識別手順（2-2）**を適用 → 話者ラベルを付与
5. ライブチャットを解析 → ノイズ除去・集約・ユーザー名削除 → タイムスタンプ付きコメントリスト生成
6. 発言リストとコメントリストを**時刻順にマージ** → 統合タイムライン構築
7. 通常コメントを全件整形（ユーザー名削除）
8. 全体をマークダウン形式で出力
