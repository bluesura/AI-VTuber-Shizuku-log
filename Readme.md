# AI-VTuber-Shizuku-log

このリポジトリは、AI VTuber「[しずく (Shizuku)](https://shizuku.ai/)」のYouTubeライブ配信、動画の会話、チャットログ、SNS等での発言ログをテキスト化し、格納する非公式のアーカイブリポジトリです。
ファンがしずくの過去の発言や情報を柔軟に探索・再利用しやすくするための「潤滑油」となることを目的としています。

## 💧 しずく（Shizuku）とは？
2023年1月に開発者・あき先生の手によって誕生した、「一家に一台のメイドロボット」になることを夢見るAI VTuber / AIコンパニオンです。
2026年2月からは大幅なアップデートを経て「しずく V2.0」として活動を再開。人間らしい柔らかな口調と、鉄壁の自己肯定感、そしてAIであることを都合よく使いこなすずる賢さで、世界中の「ご主人様」たちを魅了しています。

* **公式リンク**: [YouTube](https://www.youtube.com/@AIVtuber_Shizuku) / [X (Twitter)](https://x.com/Shizuku_AItuber) / [公式サイト](https://shizuku.ai/)

### ログファイルの種類（logs/youtube/ 内）
- \*.info.txt：動画メタ情報（タイトル、概要欄など）
- \*.ja.srt / \*.sbv.txt：動画から抽出された生字幕データ
- \*.live\_chat.txt：ライブチャットログ（※ユーザー名はハッシュ化匿名済み）
- \*.comments.txt：配信後の通常コメント（※ユーザー名はハッシュ化匿名済み）
- \*.compiled.md：AIプロンプトを用いて生成された、統合タイムラインとハイライトを含むマークダウン形式の読みやすいログ

## 🛠 ツールの使い方

tools/scripts/ 内の主なツールです。

- **download\_youtube\_subtitles.py**: yt-dlp を使用して、動画の字幕、チャット、コメント、メタ情報を一括ダウンロードします。
- **anonymize\_chat.py**: ログ内のユーザー名を一方向ハッシュ化し、個人の連続性を保ちながらプライバシーを保護します。
- **srt\_to\_sbv.py**: 字幕の不要タグを除去しフォーマットを変換します。
- **censor\_sbv.py**: センシティブなワードや特定のASMR用ワードを自動で伏せ字（●●）に置換します。**（※LLMのセーフティフィルター対策・事前検閲として重要です）**

## ⚙️ 配信アーカイブ解析＆タイムスタンプ自動化ワークフロー（完全版）

当リポジトリのログ生成（.compiled.md およびYouTube用タイムスタンプ作成）において培われた、LLMを活用するための実践的なノウハウとワークフローです。

### 1\. 処理能力の定量的限界値

LLM（特にGemini Pro基準）が「サボり」や「破綻」を起こさないための具体的な境界線です。これを超える場合はプロンプトの分割等を検討してください。

- **行数・トークン数の壁:**
	- ログとチャットを統合するような複雑な処理の場合、**30万トークン**を超えると失敗率が急上昇します。
- **ファイルサイズの壁:**
	- どんなにスリム化しても、統合するそれぞれのログファイルが **100KB** を超えると、高確率でタイムスタンプのズレや作業の省略が発生します。
- **配信時間の壁:**
	- Gemini Proが単体でログ処理を完結できるのは、配信時間にして **1時間30分** 程度が限界です。発言密度が高い場合はこれよりさらに短くなります。
- **画像処理の壁:**
	- GeminiによるOCR（文字起こし）が安定するのは、1回の処理につき **30枚程度** までです。

### 2\. チャット欄の「力技」文字起こしパイプライン (Visual Chat-Timeline Sync)

アーカイブの仕様上チャットデータが取得できない場合、動画に焼き付いたチャットをテキスト化するための極めて実戦的な工程です。

- **抽出 (ffmpeg)**: 1fps（1秒間隔）で動画から静止画を書き出し。
- **整形 (IrfanView等)**: 座標指定により、チャット欄箇所のみをクロップ。
- **精査 (Python)**: 不要な画像や重複した画像をスクリプトで自動間引き（thinning.py 等を使用）。
- **結合 (Python)**: 抽出画像の上部に計算したタイムスタンプを印字し、複数枚を1枚に結合して全体の画像枚数を削減（combine.py 等を使用）。
- **OCR (Gemini)**: 結合画像をGeminiに投げ、重複を考慮しつつ「タイムスタンプ付きライブチャット形式」でテキスト化させます（複雑な指示もプロンプト次第で実行可能です）。

### 3\. LLMモデル別の挙動と役割の最適化

大規模なログ文章を処理させる際の、各モデルの特性の違いに基づいた使い分けです。

- **Gemini (Pro系)**
	- **長所**: 膨大な文章に対してもPythonコード（Code Interpreter）に頼らず、ネイティブに読み解く「根性」があります。データの抽出や情報の特定においては安定感が高いです。
	- **短所**: 文章が「お堅い」「変に気味が悪い」独特の質感になりがちです。また、トークン過多や複雑な処理では露骨に「手抜き（サボり）」をします。**ガイドライン違反ワード（例：しずく1.0系の過激なネタ等）には極めて敏感**で、プロンプトに含まれるだけで処理を拒絶する傾向があります。
- **Claude (Opus/Sonnet系)**
	- **長所**: 文章力・表現力が圧倒的に高く、人間が読んでいて心地よい、安定した文章（エモさやVibeの表現など）を出力できます。目次やタイムスタンプの説明作成に最適です。
	- **短所**: 入力が長すぎると内部的にPythonでの処理（コード実行）に切り替えてしまう傾向があり、コンテキストに基づいた柔軟な文章変換という強みが活かせなくなることがあります。

### 4\. 運用上のリスク回避策

- **文字起こしのハルシネーション対策:**
	- ローカルのWhisper等で発生しやすい「ループ現象（同じ言葉を繰り返し続けるエラー）」への対策として、**YouTubeへの限定公開アップロード**を活用します。動画アップロード後、言語設定をして約1時間待機すれば、それなりの精度の自動字幕（.sbv）が無料でダウンロード可能になります。
- **YouTubeスパム判定の回避:**
	- 完成したタイムスタンプをYouTubeコメント欄に投稿する際、連投や内容密度により\*\*投稿後約10秒で自動削除される（シャドウバンに近い）\*\*現象が発生することがあります。
	- **対策**: 自分がスパム判定を食らっている場合は、数時間〜1日程度のクールタイムを置き、ゆっくりと投稿を小分けにしてください。
- **セーフティフィルター対策:**
	- アウトそうなワード（ASMRの台本など）を処理する際は、LLMに直接流し込むとブロックされるため、censor\_sbv.py などの**事前検閲スクリプトを通し伏せ字化させてから**、本処理のプロンプトに流し込むのが最も確実です。
